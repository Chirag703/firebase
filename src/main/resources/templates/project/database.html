Firestore Database

A flexible, scalable NoSQL cloud database for your project.

Firestore is enabled. Your app can read and write to the database.
Firestore is not enabled. Enable it in Project Settings → Services.
Data Model

Firestore organizes data into collections containing documents. Each document can have subcollections.

users/ collection
user_abc123 document
posts/ subcollection
messages/ collection
Security Rules

Current security rules for your Firestore database:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write:
        if request.auth != null;
    }
  }
}
API Reference
GET — Read Document
GET /api/v1/project/{id}/db/{collection}/{docId}
POST — Write Document
POST /api/v1/project/{id}/db/{collection}
PUT — Update Document
PUT /api/v1/project/{id}/db/{collection}/{docId}
DELETE — Remove Document
DELETE /api/v1/project/{id}/db/{collection}/{docId}
